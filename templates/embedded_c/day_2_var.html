What is Embedded Systems?

To understand Embedded C, we must first understand Embedded Systems. An Embedded System is a combination of electronic hardware usually a microcontroller or microprocessor and software interfaced with mechanical parts, sensors or actuators, to perform specific tasks. There are so many Embedded Systems around us. Our printers, refrigerators, dish washers, computer mice, microwaves among many other devices.



General Purpose Vs Dedicated Systems

General Purpose systems are systems such as your personal computer. They perform all sorts of tasks while Dedicated systems are the embedded systems described above.



Embedded Software

Embedded Software refers to the code written to interact with hardware. The code is usually written in an IDE and uploaded to the memory of the micro-controller.



Embedded C

Embedded C is a slight variation of C used to program microcontrollers and microprocessors. It is very similar to C Programming Language.

Embedded Programming Languages

There are other programming languages used programming embedded systems. Some of them are:
  • C++: Apart from C, C++ is the most widely used programming language for microcontrollers.
  • Rust:
  • Java:
  • Python: This is the most widely used programming language for microprocessors.
  • Assembly: This refers to low level programming language intended to communicate directly with the hardware.

There are other programming languages that are used for programming embedded systems. Discussing them is beyond the scope of this course.

Processors

There are several types of processors - microprocessors, microcontrollers, Digital Signal Processors(dsp), Complex Programmable Logic Device or Field Programmable Gate Arrays(CPLD/FPGA), System-on-Chip(soc). general purpose processors such as Central Processing Unit(CPU) or Graphics Processing Unit(GPU), Application Specific Integrated Circuit(ASIC).

  1. Microprocessor: A microprocessor is a data processing device that performs arithmetic and logic operations. It requires external input and output peripherals.
  2. Microcontroller: A micro-controller comprises a microprocessor and other peripherals such as memory, and Input and Output peripherals on a single chip. In a microcontroller, you cannot customize the peripherals.
  3. Complex Programmable Logic Devices: CPLDs are not as complex as FPGAs but more complex than Simple Programmable Logic Devices. They are reconfigurable. This means that the peripherals can be customised. CPLDs are control-oriented and can be used just as microcontrollers. These Devices can be programmed using Hardware Description Languages. There is no processor core on the CPLD. The parts or blocks on a CPLD are:
    - Logic blocks
    - I/O blocks
    - Clock routing
    - Routing matrix
    - Memory: Non-volatile EEPROM memory
  4. Field Programmable Gate Arrays: These are integrated circuits that can be manipulated or configured by a customer after manufacturing. This means they have customizable design to optimize design for speed and cost. These are data path-oriented and can perform complex mathematical calculations just as DSPs. FPGAs can be programmed using C programming language. FPGAs are fully parallel in implementation. The parts of an FPGA are:
    - Logic blocks
    - I/O blocks
    - Clock routing
    - Routing matrix
    - Memory
    - Multipliers/DSP blocks
    - Processor: There can be either soft or hard core on an FPGA. The soft core can be created using software while the hard core is fabricated in the lab. Software can be used to create multiple cores.

    FPGAs have certain advantages over DSPs. Since they are reconfigurable, they can take up less space on boards(PCBs).
  5. Digital Signal Processors: These are specialized microprocessors designed to processor real-world signals like voice, audio, video, temperature, pressure, or position that have been converted to Digital signals for mathematical manipulations. DSPs are specialized processors so they are sequential in nature. They go through a fetch, decode and execute cycle.
  6. Central Processing Unit: Central Processing Units are arithmetic and logic devices that power your personal computers, laptops and desktops.
  7. Graphic Processing Unit: These processors are dedicated for very large computations and can handle specific tasks like performing computations on a lot of data or graphics.
  8. System-on-Chip: Mobile phones are a great example of System-on-chip. This means having two or more on a single chip. On mobile phones, there can be different kinds of processors, CPU, GPU, DSP and etcetera. When there are identical processors it is called symmetric with one operating system and a common shared memory. A chip with different processors, instruction sets, different operating systems, and possibly without shared memory is called Asymmetric. Examples: Snapdragon, Raspberry Pi
  9. Application Specific Integrated Circuits: These are special chips that are purpose-built for an application. The function of the chip cannot be changed after it has been built.
